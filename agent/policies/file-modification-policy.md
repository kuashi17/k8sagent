# File Modification Policy

## 목적

AI Agent가 생성 또는 수정 대상 파일을 다룰 때의 기준을 정의합니다.

## 원칙

- 전체 재생성보다 부분 수정 가능 여부를 우선 검토합니다.
- 사용자 작성 변경 사항을 임의로 덮어쓰지 않습니다.
- 수정 전 대상 파일과 변경 목적을 명확히 합니다.
- 생성 산출물과 수동 작성 파일을 구분합니다.
- 검증 실패 원인이 특정 산출물에 한정되면 해당 파일만 수정합니다.
- 변경 후 관련 검증 명령을 다시 실행합니다.

## 부분 재생성 후보

- CRD 타입 정의
- Controller Reconcile 로직
- RBAC marker
- Manifest
- 테스트 초안
- README 또는 사용 문서
