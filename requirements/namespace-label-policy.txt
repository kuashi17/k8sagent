NamespaceLabelPolicy라는 Kubernetes Custom Resource를 관리하는 Operator를 만들고 싶다.

이 Operator의 목적은 대상 Namespace에 요구되는 label을 멱등적으로 반영한다.

domain은 sample.io, group은 policy, version은 v1alpha1, kind는 NamespaceLabelPolicy로 한다.

spec에는 다음 필드를 포함한다.
- namespaceName:string - label을 적용할 Namespace 이름
- labels:map[string]string - Namespace에 반영할 label key/value

status에는 다음 필드를 포함한다.
- phase:string - 정책 적용 상태
- observedNamespace:string - 마지막으로 확인한 Namespace 이름
- message:string - 현재 상태 설명 또는 오류 메시지

Controller는 NamespaceLabelPolicy Custom Resource 변경을 감지한다.
Controller는 Namespace 리소스를 조회하고 label을 수정한다.

Controller는 다음 규칙에 따라 spec 값을 관리 리소스에 반영한다.
- spec.labels -> Namespace.metadata.labels
- spec.namespaceName -> Namespace.metadata.name

Controller는 Namespace 상태를 조회하여 status를 갱신한다.
- status.phase는 요구 label이 모두 반영되었는지를 기준으로 갱신한다.

NamespaceLabelPolicy가 삭제되어도 Namespace 자체는 삭제하지 않는다.

필요한 RBAC 권한은 다음 리소스를 기준으로 추론한다.
- core/namespaces

검증 명령은 다음을 사용한다.
- make generate
- make manifests
- make test

샘플 Custom Resource는 다음 값을 사용한다.
apiVersion: policy.sample.io/v1alpha1
kind: NamespaceLabelPolicy
metadata:
  name: namespace-label-policy-sample
spec:
  namespaceName: default
  labels: {environment: development}
