SecretSync라는 Kubernetes Custom Resource를 관리하는 Operator를 만들고 싶다.

이 Operator의 목적은 사용자가 입력한 값을 Kubernetes Secret으로 관리한다.

domain은 sample.io, group은 security, version은 v1alpha1, kind는 SecretSync로 한다.

spec에는 다음 필드를 포함한다.
- secretName:string - 생성할 Secret 이름
- data:map[string]string - Secret stringData에 반영할 key/value 값
- enabled:bool - Secret 생성 활성화 여부

status에는 다음 필드를 포함한다.
- phase:string - 현재 처리 상태
- secretName:string - 생성 또는 갱신된 Secret 이름
- message:string - 현재 상태 설명 또는 오류 메시지

Controller는 SecretSync Custom Resource 변경을 감지한다.
Controller는 Secret 리소스를 생성/수정/삭제한다.

Controller는 다음 규칙에 따라 spec 값을 관리 리소스에 반영한다.
- spec.data -> Secret.stringData
- spec.secretName -> Secret.metadata.name

Controller는 Secret 상태를 조회하여 status를 갱신한다.
- status.phase는 Secret 생성/갱신 여부를 기준으로 갱신한다.

SecretSync가 삭제되면 ownerReference 정책에 따라 하위 리소스를 정리한다.

필요한 RBAC 권한은 다음 리소스를 기준으로 추론한다.
- core/secrets

검증 명령은 다음을 사용한다.
- make generate
- make manifests
- make test

샘플 Custom Resource는 다음 값을 사용한다.
apiVersion: security.sample.io/v1alpha1
kind: SecretSync
metadata:
  name: secret-sync-sample
spec:
  secretName: secret-name-sample
  data: {EXAMPLE_KEY: example-value}
  enabled: true
