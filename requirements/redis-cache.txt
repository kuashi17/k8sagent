RedisCache라는 Kubernetes Custom Resource를 관리하는 Operator를 만들고 싶다.

이 Operator의 목적은 Redis StatefulSet과 Service를 생성하고 지속 스토리지를 관리한다.

domain은 sample.io, group은 cache, version은 v1alpha1, kind는 RedisCache로 한다.

spec에는 다음 필드를 포함한다.
- size:int32 - Redis replica 수
- image:string - Redis 컨테이너 이미지
- storageSize:string - replica별 persistent storage 요청 크기
- port:int32 - Redis Service와 컨테이너 포트

status에는 다음 필드를 포함한다.
- phase:string - 현재 처리 상태
- readyReplicas:int32 - 준비된 StatefulSet replica 수
- message:string - 현재 상태 설명 또는 오류 메시지

Controller는 RedisCache Custom Resource 변경을 감지한다.
Controller는 StatefulSet, Service 리소스를 생성/수정/삭제한다.

Controller는 다음 규칙에 따라 spec 값을 관리 리소스에 반영한다.
- spec.size -> StatefulSet.spec.replicas
- spec.image -> StatefulSet container image
- spec.storageSize -> StatefulSet volumeClaimTemplates storage request
- spec.port -> StatefulSet container port and Service port

Controller는 StatefulSet 상태를 조회하여 status를 갱신한다.
- status.readyReplicas는 StatefulSet.status.readyReplicas를 기준으로 갱신한다.

RedisCache가 삭제되면 ownerReference 정책에 따라 하위 리소스를 정리한다.

필요한 RBAC 권한은 다음 리소스를 기준으로 추론한다.
- apps/statefulsets
- core/services
- core/persistentvolumeclaims

검증 명령은 다음을 사용한다.
- make generate
- make manifests
- make test

샘플 Custom Resource는 다음 값을 사용한다.
apiVersion: cache.sample.io/v1alpha1
kind: RedisCache
metadata:
  name: redis-cache-sample
spec:
  size: 1
  image: redis:7
  storageSize: 1Gi
  port: 6379
